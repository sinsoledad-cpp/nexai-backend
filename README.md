# 一、 简历智能解析 (Smart Parsing)
解析是整个流程的基石，目标是将非结构化的 PDF/Word 转换为标准化的结构化数据（JSON）。

## 功能设计：
- 多格式支持： 兼容 PDF、Docx、图片（OCR）。
- 实体提取： 精准识别个人信息、教育背景、工作经历（公司、职位、起止时间）、项目经验、技能栈。
- 意图修正： 当解析出现偏差时（如将项目名识别为公司名），允许用户在预览界面手动修正。

## 技术实现：
- 解析链路： 采用“规则过滤 + LLM 结构化提取”的混合方案。先用 Golang 的 pdf-cpu 或 document-parser 提取纯文本，再通过 LLM 的 Function Calling 或 Structured Output 特性输出 JSON。
- Eino 应用： 构建一个解析图 (Graph)，节点包括：文本清洗 -> 模块切分 -> LLM 提取 -> Schema 校验。

# 二、 AI 简历评分系统 (AI Scoring System)
评分不应只是一个模糊的分数，而应具有权威性和可解释性。

## 功能设计：
- 多维维度： 完整度（联系方式是否缺失）、专业度（是否使用行业术语）、量化度（是否有 STAR 法则中的数据支撑）、排版视觉。
- 行业基准： 根据用户投递的意向岗位（如 Backend vs. UI/UX）动态调整评分权重。

## 技术实现：
- 评分 Agent： 编写不同的评估插件。例如，一个插件专门检测“量化指标”，通过正则表达式和 LLM 扫描经历中是否包含百分比、金额或时间缩短等关键词。
- 数据聚合： 应用层汇总各维度的评分和理由，生成一份 JSON 报告，前端渲染为环形图或雷达图。

# 三、 简历优化建议 (Optimization Suggestions)
这是 AI 价值最直接的体现，将发现的问题转化为可执行的操作。

## 功能设计：
- 缺陷诊断： 识别“描述过于平淡”、“缺乏核心关键词”、“段落过长”等问题。
- STAR 改写建议： 自动为每一条工作经历提供基于“情境(S)、任务(T)、行动(A)、结果(R)”的改写预览。
- JD 匹配度优化： 用户上传目标职位的描述 (JD)，AI 分析简历与 JD 的间隙 (Gap)，建议增加哪些关键词。

## 技术实现：
- Contextual Prompting： 在 Prompt 中引入行业知识库。例如，如果是 Golang 后端简历，提示词应要求 AI 重点关注“并发处理、中间件优化、DDD 实践”等描述。
- Diff 输出： 后端返回建议时，不仅返回文本，还要返回建议类型（增加/修改/删除），方便前端做 UI 上的差异展示。

# 四、 AI 增强编辑器 (AI-Enhanced Editor)
这是用户停留时间最长的地方，需要极致的交互体验。

## 功能设计：
- Slash Command (斜杠命令)： 输入 / 唤起菜单，如“/润色这段文字”、“/增加量化数据”。
- 流式补全 (Autocomplete)： 类似 GitHub Copilot，在用户撰写工作内容时，根据上下文进行实时续写。
- 动态侧边栏： 侧边栏实时同步 AI 建议，点击建议卡片上的“一键应用”，编辑器内容自动替换。

## 技术实现：
- 前端选型： 推荐使用 Tiptap (基于 ProseMirror)，它对自定义节点和行内交互支持最好。
- 后端流式传输： 采用 Server-Sent Events (SSE) 或 WebSocket。Golang 后端通过 Chan 接收模型生成的 Token，并实时推送到前端。
- 版本协同： 使用 OT 或 CRDT 算法确保 AI 自动修改与用户手动编辑不会产生冲突。
